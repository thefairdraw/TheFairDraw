FAQ

❓ What is an “ephemeral seed”?

It is a randomly generated 64-character string created at the moment of the draw.
It is never reused or stored.
It must be preserved by the user to allow future verification.

❓ How can I verify a past draw result?

Use the list, the ephemeral seed.
Enter them into the verifier tool at thefairdraw.com
The system will re-compute the result and confirm whether it matches.

❓ Can results be tampered with?

No. The output is deterministic and hash-based.
If any input changes, the result changes.
Verification logic is public and reproducible.

❓ Is the random function cryptographically secure?

Yes. The system relies on SHA-256, a standard cryptographic hash function.
It does not use non-deterministic functions like Math.random() or system entropy sources.

❓ Why is there no blockchain or consensus mechanism?

Because this is a stateless protocol.
There is nothing to synchronize or store.
All logic runs client-side, and fairness is guaranteed by hash integrity, not a network of nodes.

❓ What is the maximum list size?

1,000,000 entries per draw.
Larger sets require custom deployment. You’re free to fork the open-source code for that.

❓ Can I use this for lotteries or voting?

Yes — technically.
The system offers unbiased selection, but legal or regulatory use depends on your jurisdiction.
TheFairDraw does not enforce or monitor usage.

❓ Do I need an API key or registration?

No. The service is fully open.
You can integrate or access it without any credentials.

❓ Is the code audited?

It is publicly available and open for peer review.
Formal third-party audits are not currently planned.

❓ Where can I find the source code?

github.com/thefairdraw
The entire logic, including the randomness generator and verifier, is licensed under MIT.

❓ How does this differ from Chainlink VRF or other oracles?

TheFairDraw does not use external randomness providers or oracles.
It prioritizes transparency and determinism over non-deterministic entropy.
This makes it suitable for stateless applications.
